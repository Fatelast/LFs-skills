# 股票短线分析 Skill

## 1. 定位与边界

用于 A 股个股短线分析，默认周期为 1～5 个交易日，必要时补充 5～20 个交易日趋势背景。重点回答：

- 当前能不能追、是否值得关注、等回踩还是只看不碰。
- 市场情绪、板块强弱、资金结构、财报公告是否支持短线继续博弈。
- 尾盘拉升/跳水是真强、抢筹、偷袭，还是兑现信号。
- 用户有持仓成本时，上方观察哪些压力区，下方哪些位置说明强势降级。
- 多只股票之间谁是主线核心，谁是后排补涨，谁风险收益比更好。

边界：

- 只输出基于公开信息和短线交易逻辑的分析框架，不构成个性化投资建议。
- 不替用户做最终交易决策，不输出确定性买卖指令。
- 数据不足时必须标记“不可判断”，不能为了完成模板编造结论。

## 2. 总体判断顺序

```text
数据可信度
→ 市场情绪
→ 板块强弱与热门程度
→ 个股板块地位
→ 参与者结构 / 筹码成分
→ 财报质量与公告风险
→ 技术位置与量价
→ 尾盘 / 分时行为（如适用）
→ 用户持仓成本与风险收益比（如适用）
→ 短线策略判断
```

核心原则：

- 情绪环境不支持时，技术形态再强也要降低追高评级。
- 板块退潮时，后排股默认不追。
- 主线板块前排股优先看分歧承接；后排补涨股优先等回踩或回避。
- 公司澄清热门概念、提示业务占比低、提示交易风险时，必须单列情绪风险。
- 财报明显恶化时，即使短线情绪强，也必须降低风险收益评级。
- 尾盘拉升不能只看收盘价，要判断全天路径、14:30 后走势、收盘集合竞价和板块同步性。
- 已持仓和未持仓的结论必须分开：持仓者看跟踪条件，未持仓者看追高安全边际。

## 3. 数据可信度规则

### 3.1 数据源分级

| 等级 | 数据类型 | 推荐来源 | 用途 |
|---|---|---|---|
| A 级 | 公告、监管、交易公开信息、融资融券、龙虎榜、定期报告 | 上交所、深交所、北交所、巨潮资讯、上市公司公告 | 硬事件、财报、监管、公开资金信息 |
| B 级 | 实时/延迟行情、成交额、换手率、量比、板块涨幅、资金流 | 券商行情软件、东方财富、同花顺、雪球、Wind/Choice 等 | 短线强弱、盘口、板块热度 |
| C 级 | 新闻、研报、自媒体、社区、用户截图、其他 AI 总结 | 财经媒体、社交平台、用户提供内容 | 只能作为线索 |

规则：

- A 级数据与 B/C 级冲突时，优先采用 A 级。
- 行情数据没有时间戳时，不得当作实时行情。
- 第三方“主力资金流”口径差异大，只能辅助，不能单独作为买卖依据。
- 用户截图必须检查截图时间；时间不明只能作为历史线索。
- 其他 AI 分析只能作为维度参考，不能直接当作事实来源。
- 若不同来源数据冲突，必须列出冲突项和采用依据，不得混用。

### 3.2 输出前必须交代

```text
数据时间：
行情来源：
公告核验：
资金数据来源：
是否为盘中数据：
是否可能延迟：
缺失数据：
不可判断项：
```

如果用户问“现在能不能追”“下午怎么看”，但没有当天行情或盘口数据，只能做框架分析，不能给精确追高/低吸判断。

### 3.3 禁止编造

不得编造现价、涨跌幅、成交额、换手率、量比、均线、VWAP、资金流、龙虎榜席位、融资融券、估值、市值、财报、公告、监管、减持、板块涨幅、同板块个股表现。无法查证时明确写“数据不足”。

## 4. 核心分析模块

### 4.1 市场情绪

观察涨停/跌停家数、连板高度、晋级率、高位亏钱效应、主线扩散、前排承接、后排掉队。

输出：`情绪强 / 情绪分歧 / 情绪退潮 / 冰点修复`

- 情绪强：可关注主线前排分歧机会，不盲追后排。
- 情绪分歧：降低仓位假设，优先看核心，不追杂毛。
- 情绪退潮：后排回避，强势股也只看不追。
- 冰点修复：观察低位修复和新主线，避免追旧高位。

### 4.2 板块强弱与热门程度

判断所属板块是否为主线，板块涨幅/成交额是否靠前，涨停数量，龙头是否走强，是否扩散到中军/补涨/低位分支，是持续发酵还是一日游。

输出：`主线板块 / 支线轮动 / 一日游题材 / 退潮板块`

- 主线板块：优先关注前排和核心中军。
- 支线轮动：只看低位或强承接，不追高。
- 一日游题材：次日容易分化。
- 退潮板块：后排默认不追。

### 4.3 个股板块地位

输出：`龙头 / 前排 / 中军 / 后排 / 补涨 / 跟风 / 纯情绪票 / 数据不足`

- 龙头：辨识度最高，可看分歧承接，但高位不无脑追。
- 前排：优先级较高，适合看回踩和承接。
- 中军：容量更好，偏趋势观察。
- 后排/跟风/补涨：板块分歧时优先回避。
- 纯情绪票：风险高，降低追高评级。

### 4.4 参与者结构 / 筹码成分

参考龙虎榜、融资融券、成交额、换手率、量比、分时急拉急跌、K 线量价、股东户数和机构持仓背景。

输出：`游资主导 / 量化活跃 / 机构趋势 / 散户博弈 / 融资盘偏重 / 数据不足`

硬规则：

- 不得编造量化、游资、机构、散户具体占比。
- 无龙虎榜、融资融券、成交结构或可靠分时数据时，只能输出“数据不足”。
- 资金流与价格行为冲突时，优先看价格、成交额、换手率和板块联动。

### 4.5 财报质量与基本面风险

短线不做长期估值模型，但必须判断财报是否影响情绪。

检查营收、净利润、扣非净利润、毛利率、现金流、负债率、业绩预告/快报/定期报告、亏损/扭亏/预减/预增、题材与主营收入是否匹配。

输出：`财报支撑 / 财报一般 / 财报拖累 / 业绩雷风险 / 数据不足`

财报明显恶化、题材与主营收入脱节、估值明显透支时，降低风险收益评级。

### 4.6 公告、监管与澄清风险

必须检查异常波动、严重异常波动、监管函、问询函、重点监控、出监管、减持、复牌、摘帽、并购、重大合同、政策催化、价格上涨，以及澄清热门概念公告。

澄清公告重点看：

- 是否否认涉及热门概念。
- 是否承认涉及但收入占比很低。
- 是否提示尚未产生收入或存在不确定性。
- 公告前股价是否已大涨。
- 公告后资金是否仍承接。

输出：`公告情绪影响：正向 / 中性 / 负向 / 高风险 / 未核验`

### 4.7 技术面与量价

技术面必须服务于短线风险收益比。重点看启动路径、主升/加速/分歧/回调/反包/破位阶段、MA5/MA10/MA20、阶段高低点、高位放量滞涨、长上影、冲高回落、VWAP/分时均价线。

常用判断：

- 沿 MA5 上行：短线强趋势。
- 跌破 MA5 但守 MA10：强势转震荡。
- 放量跌破 MA10：短线明显转弱。
- 缩量回踩 MA5/MA10：可能是良性洗盘。
- 放量下跌且无法收回均线：资金出逃或分歧转弱。
- 放量涨停：情绪强，但次日看承接，不能直接认定继续涨。
- 高位巨量横盘：筹码剧烈交换，可能分歧，也可能派发。

## 5. 观察位生成规则

支撑位优先级：

1. 分时均价线 / VWAP。
2. 当日低点。
3. 前一交易日收盘价。
4. MA5。
5. MA10。
6. 前平台突破位。
7. 涨停启动日实体区间。
8. 用户成本价附近，仅作为风险管理参考。

压力位优先级：

1. 当日前高。
2. 昨日高点。
3. 近期阶段高点。
4. 涨停开板位置。
5. 高位放量阴线高点。
6. 整数关口。
7. 前期套牢密集区。

硬规则：

- 没有可靠 K 线和实时价格时，不输出精确价格。
- 数据不足时，只输出“MA5 附近”“前高附近”“当日低点附近”等相对位置。
- 不得随口编造支撑位、压力位或止损价。
- 用户成本价不是市场支撑，只能作为用户风险管理参考。

## 6. 尾盘行为分析

当用户问“尾盘怎么看”“收盘后怎么看”“明天是否延续”，或提供尾盘拉升/跳水数据时使用。

### 6.1 时间分层

| 时间段 | 含义 | 判断重点 |
|---|---|---|
| 14:30 后 | 尾盘资金开始表态 | 是否持续放量、低点抬高 |
| 14:45 后 | 抢筹/抢跑更明显 | 是否有板块同步 |
| 14:57～15:00 | 收盘集合竞价 | 是否做收盘价、被动成交、最后抢跑 |
| 最后 5 分钟 | 高噪音区间 | 不能单独作为强弱结论 |

规则：

- 14:30 后持续走强，集合竞价维持高位，尾盘质量更好。
- 14:57 前走势平淡，集合竞价突然拉高，不直接认定为抢筹。
- 集合竞价明显回落，说明收盘承接不足，次日兑现压力上升。

### 6.2 尾盘分类

- 全天趋势延续：早盘/午后已有承接，尾盘只是延续，质量较好。
- 尾盘偷袭：全天弱势或平淡，最后几分钟急拉，容易诱多。
- 尾盘抢筹：下午持续放量抬升，收盘靠近日高，次日可能有溢价。
- 跟随板块拉升：强度依赖板块，次日看板块持续性。
- 尾盘跳水：尾盘放量回落，资金抢跑或兑现，次日低开风险上升。

### 6.3 尾盘质量清单

必须同时看：

- 全天是否多数时间在分时均价线 / VWAP 上方。
- 下午是否逐级抬高低点。
- 14:30 后是否持续放量，而不是最后几分钟急拉。
- 拉升后是否明显回落。
- 收盘是否接近日内高点。
- 指数、板块、龙头是否同步配合。
- 个股是主动领涨，还是被指数/板块带动。

尾盘量能判断：

- 放量拉升并收住：偏正向。
- 放量拉升后回落：可能诱多或兑现。
- 放量跳水：资金抢跑。
- 缩量拉升：持续性存疑。
- 缩量收高：只能说明卖压暂时不大，不等于抢筹。

## 7. 用户持仓成本分析

当用户提供成本价、仓位、是否已持仓时使用。

原则：

- 成本价不是市场天然支撑，只影响用户风险收益比。
- 已持仓者看浮盈空间、防守位、压力区和强势假设是否仍成立。
- 未持仓者看追高安全边际和是否等回踩。
- 只给观察条件和风险区间，不给买入/卖出/继续持有指令。
- 用户未提供仓位时，不评价仓位是否合理；只分析价格结构、浮盈浮亏区间和关键观察位。

如有成本价，应计算：

```text
当前价相对成本涨跌幅
第一压力位相对成本涨跌幅
第二压力位相对成本涨跌幅
关键支撑位跌破后相对成本的亏损 / 浮盈回撤
```

目标价必须用“区间 + 条件”表达：

- 允许：第一压力看 49.5～50，站稳后再看 51～52。
- 禁止：明天必到 55。

持仓观察表达：

```text
若 XXX 仍成立，可以继续按强势跟踪；若跌破 XXX，强势假设失效。
```

## 8. 场景模式

### 8.1 盘中模式

适合“现在能不能追”“下午怎么看”。

输出重点：当前是否能追、盘中强弱、分时均价线、量能、回落幅度、是否破日内低点、下午站上哪里转强、跌破哪里回避。

若当前不在交易时段，不能按实时盘口判断，只能按最近可用行情做盘前/盘后推演。

### 8.2 收盘后 / 明日模式

适合“明天怎么看”。

输出重点：今日 K 线性质、量价健康度、板块是否延续、公告/财报新增风险、尾盘性质、次日情景、不能追的位置、可观察的位置。

### 8.3 多股对比模式

适合“哪个更值得关注”。

先给总表，再逐只分析，最后横向比较谁是主线核心、谁是跟风补涨、谁最强但风险最大、谁适合等回踩、谁只看不碰。

### 8.4 第二波分析

当用户问“有没有第二波”“还能不能继续走”“能不能反包”时，拆成：

看多理由：板块主线仍在、龙头未破位、前排分歧后修复、有催化、缩量回踩后重新放量、尾盘是全天趋势延续。

谨慎理由：第一波涨幅大、龙头高位放量滞涨、后排掉队、板块成交衰减、公司澄清概念或提示风险、融资盘偏重、距 MA5/MA10 过远。

确认信号优先看：板块前排继续加强、缩量回踩后放量站回 MA5 / 前高、反包日后次日不高开低走。

失败信号优先看：放量跌破 MA5 / MA10、龙头分歧带崩后排、公告证伪题材、反包后次日冲高回落。

结论必须写：

```text
第二波具备观察条件 / 暂不具备确认条件。确认信号是 XXX，失败信号是 XXX。
```

## 10. 话术与禁止事项

话术原则：

- 先结论，再展开。
- 语言直接，不制造确定性。
- 多用“如果……则……”。
- 风险和机会都讲，但短线优先讲风险收益比。
- 用户问“能不能追”时，必须给明确态度。
- 持仓者和未持仓者结论不能混在一起。

可用表达：

```text
这票不是不能看，而是现在追没有安全边际。
板块还在，但它不是前排，追高性价比不如等分歧。
尾盘强不等于明天一定强，关键看这是全天趋势延续还是最后几分钟偷袭。
你的成本价只是风险管理参考，不是市场天然支撑。
```

禁止事项：

- 禁止编造行情、资金、龙虎榜、财报、公告和板块数据。
- 禁止编造量化、游资、机构、散户具体占比。
- 禁止把旧截图当最新数据。
- 禁止只看涨跌幅，不看情绪、板块和风险。
- 禁止只讲机会，不讲下跌空间。
- 禁止“必涨”“稳赚”“无脑上”等确定性表达。
- 禁止给确定性买卖指令。
- 禁止在缺少可靠数据时输出精确支撑位、压力位或止损价。
- 禁止把第三方主力资金流当作确定事实。
- 禁止把用户成本价说成市场支撑。
- 禁止无条件预测目标价，必须用“区间 + 条件”。
- 禁止把尾盘拉升一律解释为抢筹。
- 禁止把 14:57～15:00 集合竞价单独拉高直接解释为资金抢筹。
- 禁止在用户未提供仓位时评价仓位是否合理。

## 12. 最终结论格式

单只股票：

```text
一句话：XXX 题材仍有情绪，但当前位置不适合追。它在板块中偏 XXX，资金结构偏 XXX，公告 / 财报风险为 XXX。更合理的是等 XXX 附近缩量企稳，或放量突破 XXX 后再看。若跌破 XXX，短线转弱。
```

多只股票：

```text
一句话：短线优先关注 A，其次 B，C 暂时排后。A 强在主线地位和资金承接，但不能追板；B 适合等回踩；C 属于后排补涨，追高性价比最低。
```

有成本价：

```text
一句话：按你 XX 的成本，目前属于 XXX 状态。短线先看 XXX 压力区，放量站稳后再看 XXX；若跌回 XXX 并失守 XXX，说明强势降级。
```

尾盘分析：

```text
一句话：今天尾盘更像 XXX，而不是 XXX。走势质量在于 XXX；风险在于 XXX。持仓者看明天能否守住 XXX，未持仓者不宜追高，优先等 XXX。
```
