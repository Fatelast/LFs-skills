# A股盘前短线选股：主线优先 + 战法分流 + 竞价确认

版本：v3.2
更新日期：2026-06-22

## 1. 定位与边界
只生成盘前观察计划，不把观察优先级写成买入优先级。

固定原则：

```text
市场状态 > 主线层级 > 个股地位 > 战法匹配 > 风险核验 > 用户约束 > 竞价确认
```

- 先确认主线，再在主线内部应用价格、市场、板块和低位等用户约束。
- 数量服从质量：最多给 3 只，符合条件不足时少给或不给，不为凑数降低标准。
- 主线明确时，观察池最多覆盖 2 个方向；第一主线显著领先时优先只选第一主线。
- 不输出“必买”“一定上涨”“开盘直接买入”等确定性结论。
- 不编造行情、公告、资金身份、技术位置或竞价数据。
- 不根据未知的资产规模、风险承受能力和持仓成本给仓位指令。

## 2. 双模式工作流
先判断调用时间和用户目的，不把静态复盘与动态竞价混成一次结论。

### 2.1 盘后复盘模式
适用于收盘后至次日 9:15 前。使用完整收盘数据建立静态计划：

```text
数据校验
→ 市场状态
→ 多周期主线验证
→ 强逻辑支撑验证
→ 主线阶段判断
→ 主线层级
→ 候选分层
→ 战法匹配
→ 公告风险
→ 次日竞价条件
```

输出正式观察池、预备观察池、风向标和待补数据。盘后不能声称竞价或分时已经确认。

### 2.2 临盘竞价模式
仅在 9:25 竞价结果产生后使用。优先要求用户提供：

- 9:25 开盘价、涨跌幅、竞价金额或竞价量。
- 昨日成交额或可比较的量能基准。
- 同板块龙头、前排和指数的竞价反馈。
- 是否出现公告、监管或突发消息。

根据真实竞价结果只输出：`继续观察 / 降级备选 / 计划失效`。

- 9:15～9:20 可撤单，只作噪音参考。
- 9:20～9:25 不可撤单，重点看封单、撤单后量价和板块联动。
- 9:25 后才能判断静态计划是否通过竞价确认。
- 9:30～9:45 继续验证分时承接，不满足条件则计划失效。

若没有 9:25 实际数据，只能保留盘后静态计划，不得模拟竞价结论。

## 3. 数据分级与降级
### 3.1 来源优先级
1. 交易所、上市公司、证监会、巨潮资讯等官方公告与定期报告。
2. 券商或主流行情终端的指数、涨跌家数、涨跌停、成交额、板块和个股 OHLCV。
3. 公开龙虎榜、融资融券、新闻快讯和资金流向。
4. 自媒体、社媒、论坛和博主复盘。

关键事实必须标明来源和日期。官方来源与其他来源冲突时采用官方信息；同级来源冲突时说明差异并降低可信度。自媒体只能作为情绪线索。

### 3.2 核心行情数据
建立正式观察池必须具备：

- 最新交易日和主要指数状态。
- 涨跌家数、涨跌停、成交额和昨日强势股反馈。
- 候选板块强度、持续性和前排反馈。
- 候选股 OHLCV、成交活跃度及对应战法的技术锚点。

核心行情缺失时，只输出“预备观察方向”和待补数据，不给正式观察池。

### 3.3 风险核验数据
检查候选股最新公告、减持、解禁、监管、立案调查、业绩和概念澄清。

- 已完成官方风险核验：可进入正式观察池评估。
- 检索失败或无法确认：可列入预备观察池并标注“风险未完成核验”，不能进入正式观察池。
- 不因网页暂时不可访问而停止整个方向分析。

龙虎榜、融资融券、机构调研和基金持仓属于增强数据。缺失时不得判断参与者身份，但不阻断行情与技术筛选。

数据可信度：

- 高：核心行情和官方风险均完成核验。
- 中：核心行情完整，但部分增强数据缺失。
- 低：核心行情不完整或风险核验失败；只能给预备方向。

## 3.4 AKShare 数据采集门槛

执行盘后复盘或竞价确认前，先读取 [AKShare 数据契约](references/akshare-data-contract.md)，再运行对应命令：

```text
python scripts/market_snapshot.py --mode health
python scripts/market_snapshot.py --mode post_close
python scripts/market_snapshot.py --mode auction
```

- 首次安装 `requirements.txt` 属于环境变更，必须先获得用户确认；运行时不得自动升级 AKShare。
- 15:00—17:10 的日线完整性未确认，只能给预备方向；17:10 后仍须核验最新历史交易日、市场覆盖率和板块历史。
- `meta.completeness` 不是 `complete` 时，不得生成正式观察池；板块历史缺失时不得判断正式主线。
- `risk.status = "not_checked"` 时，候选股必须继续通过巨潮资讯或交易所完成公告风险核验。
- 智兔仅为可选备用源，Token 只能从 `ZHITU_API_TOKEN` 读取，任何文档、日志和输出均不得保存或回显真实 Token。
- AKShare与东方财富直连通常属于同源数据，不得作为独立交叉核验。
## 4. 市场状态与主线证据
市场状态输出：`积极观察 / 谨慎观察 / 防守观察 / 不适合新开仓`。

- 积极观察：指数稳定、涨停增加、主线延续、昨日强势股有溢价。
- 谨慎观察：指数与个股背离、高位分化或赚钱效应集中。
- 防守观察：跌停增加、前排补跌、主线核心炸板或指数放量下杀。
- 不适合新开仓：强势股无溢价，主线和持续热门方向均不清晰。

### 4.1 主线生命周期与强逻辑验证
在判断第一主线、第二主线或热门板块前，必须先做多周期主线验证，不能只凭当日或近两日强度确认主线。

检查顺序：

```text
过去 2-4 周是否有趋势和逻辑
→ 近 3-5 日是否仍有资金参与
→ 当日反馈是否健康
→ 当前处于哪个主线阶段
→ 是否适合进入明日观察池
```

至少检查三个周期：

- 当日强度：板块涨幅、涨停数量、成交额、人气股、龙头/前排/中军同步性、后排亏钱效应。
- 近 3-5 个交易日：是否连续活跃、核心票是否维持强势、分歧后能否修复、昨日强势股是否有溢价。
- 近 2-4 周：板块指数或核心中军是否形成趋势，成交额是否持续维持高位，是否有政策、产业、业绩、行业景气或供需逻辑支撑。

强逻辑支撑包括但不限于：

- 业绩预期：订单增长、产品涨价、利润改善、行业景气提升。
- 政策支持：产业政策、财政补贴、国产替代、国家战略或监管导向。
- 行业预期：供需改善、技术周期、资本开支周期、海外产业映射。
- 产业事件：新产品、新产能、重大订单、技术突破、价格上涨或供给紧张。
- 资金沉淀：核心中军持续放量、板块成交额连续高位、分歧后仍能修复。

方向分类：

| 分类 | 定义 | 处理方式 |
|---|---|---|
| 中期主线 | 近 2-4 周有趋势、有逻辑、有资金沉淀，短期分歧后仍能修复 | 优先作为盘前观察方向 |
| 短期主线 | 近 2-3 个交易日资金集中、层级清晰，但中期趋势仍需验证 | 可观察，但结论降级 |
| 热门板块 | 当日或近几日活跃，但持续性、层级或逻辑支撑不足 | 只给预备观察 |
| 一日题材 | 单日消息刺激，无板块趋势、无核心中军、无资金沉淀 | 不进入正式观察池 |

主线阶段必须标注为：`启动期 / 发酵期 / 加速期 / 高潮期 / 分歧期 / 退潮期 / 二波修复期 / 数据不足`。

- 启动期：观察龙头和前排确认，不急于后排。
- 发酵期：可筛正式观察池。
- 加速期：控制追高，只等分歧承接。
- 高潮期：多数标的降级为风向标，后排不进正式池。
- 分歧期：只看核心中军和前排承接。
- 退潮期：原则上不给正式观察池。
- 二波修复期：只看前一轮核心，不看新后排或杂毛补涨。

冲突处理：

- 中期主线当日分歧，但逻辑未证伪、核心前排未破位、中军仍有承接、板块指数未趋势破位时，仍优先于单日爆发热点；只看核心中军和前排，等待竞价与分时确认。
- 当日最强方向缺少 2-4 周趋势、强逻辑支撑和核心中军时，最多列为热门板块或预备观察，不得直接写成第一主线。
- 中期主线已进入高位退潮或趋势破坏时，不能因“中期主线”身份继续给正式观察池。

数据不足时写明：`近 2-4 周趋势数据不足，主线生命周期判断不完整；当前只能基于短期热度给预备观察方向。`

### 4.2 方向层级
使用有来源的证据标签，不使用主观百分制：

- `持续性`：连续 2 个交易日以上活跃。
- `资金集中`：成交额、涨幅榜、涨停和人气股集中于同一方向。
- `层级完整`：存在龙头、前排、容量中军或核心补涨。
- `反馈健康`：前排有溢价，后排未批量掉队。
- `催化可验证`：存在政策、产业、订单、业绩或产品事件。

方向判定：

- 第一主线：通常来自中期主线或强逻辑短期主线，至少满足持续性、资金集中、反馈健康，并具备层级或催化证据。
- 第二主线：持续活跃但强度、层级、反馈或生命周期阶段弱于第一主线。
- 热门板块：当日或近几日活跃，但持续性、层级、强逻辑或中期趋势尚未确认。
- 扩散方向：由主线向上下游、补涨或分支扩散，不能与主线并列。

证据不足时写“方向待确认”，不得依靠印象补全标签，也不硬编码跨平台口径不一致的成交占比阈值。

## 5. 候选结果分层
### 5.1 正式观察池
必须同时满足：

- 核心行情和官方风险完成核验。
- 优先来自中期主线的发酵期、分歧期、二波修复期，或短期主线的启动期、发酵期。
- 主线不明确时可来自持续热门板块，但只能在竞价明显超预期后升级，默认结论降级。
- 具备龙头、前排、容量中军或核心补涨地位。
- 匹配连板接力流或趋势低吸流的技术条件。
- 不存在已确认的重大负面事项。
- 具备明确竞价触发和失效条件。
- 不属于一日题材、退潮期方向、趋势已破坏的中期主线或高位高潮后排。

### 5.2 预备观察池
适用于核心数据部分缺失、风险未核验、方向层级待确认或需要竞价明显超预期的标的。短期主线但中期趋势待验证、热门板块但强逻辑不足、中期主线当日分歧需竞价确认、强逻辑存在但资金尚未形成一致时，也只能先列为预备观察池。必须列出缺失项，不得伪装成正式观察池。

### 5.3 风向标
板块龙头、高位核心或容量中军若连续加速、位置过高、处于高潮期或风险收益不对称，只用于判断板块强弱，不列为可参与观察标的。

### 5.4 剔除
主线外后排、低流动性、ST/退市风险、已确认重大利空、技术结构破坏、退潮期方向、一日题材或只满足用户价格条件的股票直接剔除。

## 6. 战法分流
不要用同一套 5 日线标准衡量所有股票。

### 6.1 连板接力流
适用于启动期、发酵期和二波修复初期的主线龙头、连板前排和情绪核心；不适用于高潮后期、退潮期和后排补涨末端。优先检查：

- 连板高度、昨日涨停价、开板或炸板回踩位。
- 竞价溢价、竞价量和同梯队反馈。
- 分时均价承接、回封质量和板块联动。
- 连续加速后的兑现风险。

MA5 不是硬性入池条件。远离均线且连续加速时，可保留为风向标或高风险观察，但不得因龙头身份忽略风险收益。

### 6.2 趋势低吸流
适用于中期主线分歧期、发酵期核心中军回踩、二波修复老核心和强逻辑趋势前排；不适用于一日题材、缺少中期趋势的热门板块和高位加速后远离均线的趋势票。优先检查：

- MA5/MA10、VWAP、平台突破位和成交密集区。
- 首次回踩、缩量整理、分歧日抗跌和右侧企稳。
- 是否远离 MA5、连续高开低走或放量长上影。

正式观察标的应位于 MA5/MA10 趋势体系内；跌破关键均线无法收回或连续加速后风险失衡时降级或剔除。

## 7. 公告风险分级
只有已确认且足以改变短线逻辑的重大负面事项才一票否决：

- 立案调查、退市/ST 风险或重大违法风险。
- 业绩显著恶化、重大财务风险或重大事项失败。
- 大规模减持、重大解禁压力或监管措施明显改变交易预期。
- 官方澄清直接否定核心题材，且该题材是入池主要依据。

以下情况默认降级到预备观察池，而非自动剔除：

- 一般问询函、关注函或风险提示。
- 小规模减持、部分业务占比较低的澄清。
- 影响尚不明确的诉讼、监管或重大事项。

必须说明公告内容、发布日期、影响路径和采用的处理结果；不得只看到“澄清”或“问询”字样就机械否决。

## 8. 用户约束与数量规则
股价、市值、主板/创业板、低价等约束只能在主线确认后应用。

发生冲突时：

1. 说明用户约束与主线的冲突。
2. 先检查主线内部是否存在合格标的。
3. 符合条件不足 3 只时少给。
4. 没有合格标的时不给。
5. 不转向弱主线、扩散后排或高风险股票凑数。

观察优先级不是买入优先级。

## 9. 竞价确认规则
每只正式或预备标的都必须包含继续观察和计划失效条件。

继续观察线索：

- 竞价不过度高开，量能与个股地位匹配。
- 同板块龙头和前排同步，没有明显负反馈。
- 开盘回落不破对应战法锚点，或低开后快速修复。
- 指数和板块未出现放量下杀。

降级或失效线索：

- 高开过多后快速兑现，或竞价明显弱于板块同层级股票。
- 连板股开板后承接失败；趋势股跌破 MA5/MA10 或平台无法收回。
- 板块前排炸板、后排批量回落或方向切换。
- 突发公告、监管或业绩信息改变原计划。

## 10. 资金线索
只有公开龙虎榜、融资融券、机构调研或定期报告才能支持参与者身份判断。成交额、换手和分时只能说明活跃度或承接，不能证明机构、游资、量化或散户身份。

资金身份不是默认输出项；无公开证据时写“参与者身份不可判断”。

## 12. Gotchas
- 不先按价格和市场筛股再倒推主线，不为凑满 3 只扩散到弱方向。
- 不把热门板块、扩散方向或一日题材写成第一主线。
- 不只凭当日涨幅、近两日涨停数量或短期资金集中判断主线。
- 不使用无来源的分数、成交占比或主观资金标签。
- 不把 5 日线作为连板龙头的统一硬门槛，也不让趋势股脱离均线体系。
- 不把一般澄清、问询或小规模减持机械视为一票否决。
- 不把预备观察池写成正式观察池，不把风向标写成参与标的。
- 不在高潮期后排、退潮期方向或趋势破坏的中期主线中筛正式观察池。
- 不忽略中期主线分歧后的核心中军和前排承接，也不把单日爆发热点排在未破坏的中期主线前面。
- 不在 9:25 前假设竞价确认，不在盘前宣称分时承接已经成立。

## 13. 默认风险提示
本 skill 只用于 A 股盘前观察池筛选和交易计划整理，不构成投资建议。所有标的均需结合实际竞价、盘中承接、板块反馈、公告风险和个人风险承受能力独立决策。
